PROTOTYPE — pi-vimmode 0.10 editing semantics

Question: should selected features follow Neovim 0.11 exactly, including surprising repeat and register behavior, while keeping existing prompt-search and fixed keyword boundaries?

Decision pressure points
  • Insert-mode cursor movement starts a new dot-repeat segment.
  • p overwrites default registers; P preserves them.
  • Dot after Visual put repeats selection deletion, not inserted text.
  • gn/gN always mean forward/backward, independent of prior search direction.
  • Small-word uses pi-vimmode’s fixed [A-Za-z0-9_] keyword set; WORD means non-whitespace.
